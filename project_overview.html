Au-Park Project Overview

Au-Park는 Flutter 모바일 앱, FastAPI 백엔드, OpenCV 기반 비전 워커를 연결해 차량 등록, 소셜 로그인, 번호판 인식 입출차, 사전정산, 자동정산, 주차 현황을 처리하는 스마트 주차 시스템입니다.

4. 전체 아키텍처

1
Flutter 앱

사용자가 로그인, 차량 등록, 결제수단 등록을 수행합니다. 앱은 ApiClient를 통해 FastAPI와 통신합니다.

2
FastAPI 서버

사용자, 차량, 결제수단, 주차 세션, 결제 내역을 Firebase 또는 인메모리 저장소에 기록합니다.

3
Vision Worker

iPhone Continuity Camera 또는 RTSP/IP 카메라에서 프레임을 읽고 번호판을 OCR로 인식합니다.

4
LPR 자동 입출차

/api/v1/lpr/auto는 활성 주차 세션이 없으면 입차, 있으면 출차로 처리합니다.

5
홈 현황판 갱신

앱 홈 화면은 /api/v1/parking/sessions/active를 주기적으로 조회해 차량번호, 입차시간, 기본요금, 추가요금, 정산완료, 미정산 금액을 표시합니다.

5. 주요 기능

인증 및 사용자

  • 이메일 로그인/회원가입
  • Google, Kakao, Naver 소셜 로그인
  • 서버에서 소셜 토큰 검증 후 Au-Park 사용자 생성/조회

차량 및 결제수단

  • 사용자별 차량번호 등록
  • 결제수단 등록 화면
  • 사전정산, 자동 출차 정산, 추가요금 정산 내역 저장

LPR 입출차

  • 번호판 인식 후 서버로 lot_id, plate_number 전송
  • auto 모드에서 입차/출차 자동 판정
  • 카메라 가림 또는 번호판 이탈 후 재인식 준비 로그 출력

주차 현황 및 요금

  • 홈 화면 활성 주차 세션 표시
  • 기본요금과 추가요금 분리 표시
  • 사전정산 후 미정산 금액 차감 및 출차 시 차액 자동 결제

6. 요금 정책

구간 요금 비고
입차 후 30분 이내 0원 무료 출차
31분~60분 기본요금 1,500원 현황판에 기본요금으로 유지 표시
61분부터 10분마다 500원 현황판에 추가요금으로 누적 표시
일일 최대 15,000원 여러 날이면 일 단위 최대요금을 합산

사전정산 후에는 정산 완료 금액이 올라가고 현재 미정산 금액이 내려갑니다. 출차 시에는 총요금에서 이미 정산한 금액을 뺀 차액만 결제합니다.

13. 현재 한계와 다음 단계

  • Cloudflare Quick Tunnel은 재시작할 때 주소가 바뀌므로 장기 운영에는 named tunnel 또는 Cloud Run 같은 배포가 더 적합합니다.
  • 현재 자동결제는 서버 결제 내역 생성 중심입니다. 실제 카드 승인/출금을 위해서는 PortOne 빌링키 결제 API 호출을 서버에 연결해야 합니다.
  • EasyOCR 기반 번호판 인식은 테스트에 적합하지만, 운영 정확도를 높이려면 번호판 전용 YOLO/ONNX 검출 모델 도입이 좋습니다.
  • 주차면 감지는 빈 주차장 기준 이미지 방식이 가능하며, 야외 환경에서는 YOLO 차량 검출 + 주차면 ROI 교차 판정 방식이 더 안정적입니다.
  • 관리자 대시보드는 apps/admin_dashboard에 자리만 마련되어 있으며 추후 구현 대상입니다.
