각 API 그룹 역할은 이렇게 보면 됩니다.

auth
회원가입과 로그인 담당입니다.
사용자가 앱에서 계정을 만들거나 로그인하면 여기로 요청이 갑니다.

users
사용자 정보 관리 담당입니다.
마이페이지에서 내 정보 조회/수정, 내 차량 목록, 내 결제 내역 조회 같은 기능이 여기에 연결됩니다.

vehicles
차량 등록/조회/수정/삭제 담당입니다.
사용자가 자신의 차량번호, 차량 별명, 모델명 등을 등록하고 관리할 때 씁니다.

parking
주차장 상태 조회와 주차 이벤트 처리 담당입니다.
주차장 목록, 주차면 상태, 빈 자리 수, 입차/출차 기록 생성 같은 기능이 여기에 들어갑니다.

payments
결제수단 등록과 주차요금 결제 담당입니다.
현재는 PortOne 실제 연동 전 단계라 더미 결제 성공 처리로 되어 있고, 나중에 실제 PortOne API 호출로 바꾸면 됩니다.

logs
입출차 기록 조회 담당입니다.
마이페이지의 이용내역 화면에서 “언제 입차했고 언제 출차했는지” 보여줄 때 사용합니다.

lpr
번호판 인식 장치/OpenCV 모듈이 서버에 이벤트를 보내는 통로입니다.
예를 들어 카메라가 차량번호를 인식하면 /lpr/entry 또는 /lpr/exit로 보내고, 서버가 입출차 로그를 저장합니다.

요약하면 Flutter 앱은 주로 auth, users, vehicles, parking, payments, logs를 쓰고, 카메라/OpenCV 쪽은 주로 lpr을 쓰게 됩니다.
